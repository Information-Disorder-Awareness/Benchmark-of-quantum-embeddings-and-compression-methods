# A Joint Benchmark of Quantum Encodings and Compression Methods for VQC-Based Sentiment Analysis

This repository contains the code, results, and figures for the paper:

> **A Joint Benchmark of Quantum Encodings and Compression Methods for VQC-Based Sentiment Analysis**
> Danilo Cavaliere, Giuseppe Fenza, Łukasz Gajewski, Vincenzo Loia
> Department of Management & Innovation Systems, University of Salerno, Italy

The work presents a controlled benchmark of quantum embeddings and compression
methods for improving Variational Quantum Circuit (VQC) pipelines for Quantum
Natural Language Processing (QNLP). It disentangles two design axes that are
usually entangled in QNLP studies — the **quantum encoding** and the
**classical-to-quantum compression** — and measures how each affects downstream
sentiment-classification performance, seed stability, and training cost.

## Overview

A high-dimensional Sentence-BERT (SBERT) embedding must be squeezed into a
`2^n`-dimensional Hilbert space (or `n` rotation angles) before it can drive a
VQC. This benchmark isolates the effect of *how* that mapping is done by varying
only two factors and holding the rest of the pipeline fixed:

| Axis | Options |
| --- | --- |
| **Encoding** | Amplitude Embedding · Angle Embedding (X-axis rotation) |
| **Compression** | PCA · Word2Ket (W2K, a tensor-product / Kronecker compression) |
| **SBERT backbone** | `all-MiniLM-L6-v2` (384 d) · `all-mpnet-base-v2` (768 d) |
| **Circuit width** | `n ∈ {4, 5, 6, 7, 8, 9, 10}` qubits |
| **Seeds** | 42, 43, 44, 45, 46 |

That is `2 × 2 × 2 × 7 × 5 = 280` runs. All other components are held constant:
`StronglyEntanglingLayers` ansatz (`L = 6`), Pauli `X/Y/Z` measurement on each
qubit, a small 2-layer MLP head (hidden 32, dropout 0.1), Adam with a cosine LR
schedule, and the SST-2 binary sentiment task.

**Headline finding:** across all 280 runs, **Word2Ket consistently dominates
PCA** — higher test macro F1, substantially lower seed variance, and convergence
in roughly half the epochs — while the *encoding* choice, once isolated from
compression artefacts, makes no measurable difference. Compression, not the
encoding gate, is the primary design decision in NISQ-scale QNLP pipelines.

## Repository structure

```
.
├── benchmark.py                  # Runs the VQC experiments -> results/*.csv
├── analyze_results.py            # Aggregates results -> table CSV + paper figures
├── results/                      # Raw per-run results (8 CSVs, one per cell)
│   └── *.csv
├── analysis/                     # Derived outputs (generated by analyze_results.py)
│   ├── f1_stats.csv              # Per-(backbone, config, qubit) mean/std F1  -> Table 1
│   ├── summary_timing_stability.csv
│   └── plots/                    # Paper figures (Fig. 2-5)
│       ├── p3_f1_vs_qubits_by_sbert_h.png   # Fig. 2  F1 vs n-qubits by backbone
│       ├── s3_perf_stability_scatter_h.png  # Fig. 3  performance-stability tradeoff
│       ├── t2_log_stage_comparison.png      # Fig. 4  wall-clock per pipeline stage
```

The 8 raw result CSVs in `results/` are the source of truth: every table and
figure is computed from them, so the analysis can be reproduced without re-running
the (≈24 h) benchmark.

## Usage

### 1. Run the benchmark

`benchmark.py` is configured via constants near the top of the file (it has no
CLI flags). A single execution runs the full qubit range × seeds for **one**
(encoding, compression, backbone) cell and writes one CSV into `results/`. To
reproduce the full grid, run it 8 times, editing these constants between runs:

```python
SBERT_MODEL        = "sentence-transformers/all-MiniLM-L6-v2"  # or all-mpnet-base-v2
SBERT_TAG          = "all-MiniLM-L6-v2"
ENCODING_METHOD    = "angle"      # "angle" | "amplitude"
ANGLE_ROTATION     = "X"          # rotation axis for Angle encoding
COMPRESSION_METHOD = "pca"        # "pca"   | "word2ket"
```

```bash
python benchmark.py
```

SBERT embeddings are cached under `results/embeddings/` on first use and reused
across runs (so timing reflects the quantum pipeline only). The cache is
regenerated automatically if absent.

### 2. Generate tables and figures

```bash
python analyze_results.py            # reads ./results, writes ./analysis
# optional: python analyze_results.py --results_dir ./results --out_dir ./analysis
```

This produces:

- `analysis/f1_stats.csv` — per-(backbone, encoding, compression, qubit) mean ±
  std test macro F1 (the data behind **Table 1**).
- `analysis/summary_timing_stability.csv` — aggregated performance/stability/timing.
- `analysis/plots/` — the four paper figures (**Fig. 2–5**).

## Hardware & runtime

Experiments were run on an AMD Ryzen 9 9900X / 32 GB DDR5 / NVIDIA RTX 5070 Ti
(16 GB) workstation. A single run takes ≈4–12 minutes; the full 280-run benchmark
≈24 hours.
